# Telegram → Max Relay Bot

Полнофункциональный бот-ретранслятор сообщений из Telegram в мессенджер Max с поддержкой медиа, кнопок и админ-панели.

## ✨ Особенности

- 🔄 **Ретрансляция постов** из Telegram в Max в реальном времени
- 📸 **Поддержка медиа**: фото, видео, аудио, документы, медиагруппы
- 🎯 **Кнопки**: преобразование InlineKeyboard из TG в Max
- 🚫 **Фильтрация контента**: глобальные и локальные слова-исключения
- 👨‍💼 **Админ-панель**: управление каналами, словами, статистика
- 📊 **Статистика**: отслеживание пересланных и пропущенных постов
- 🔍 **Дедупликация**: не пересылает уже переслано сообщения
- 📝 **Логирование**: полное логирование всех операций

## 🛠️ Стек технологий

- **Python 3.11+**
- **aiogram 3.x** - Telegram Bot API
- **aiohttp** - асинхронный HTTP клиент для Max API
- **aiosqlite** - асинхронная работа с SQLite
- **python-dotenv** - управление конфигурацией

## 📦 Установка

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/telegram-max-relay-bot
cd telegram-max-relay-bot
```

### 2. Создайте виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Настройте переменные окружения

Создайте файл `.env` в корне проекта:

```env
TG_BOT_TOKEN=your_telegram_bot_token_here
MAX_BOT_TOKEN=your_max_bot_token_here
ADMIN_IDS=123456789,987654321
```

**Как получить токены:**

- **TG_BOT_TOKEN**: Напишите боту [@BotFather](https://t.me/botfather) в Telegram
- **MAX_BOT_TOKEN**: Получите в админ-панели Max
- **ADMIN_IDS**: Ваш Telegram ID (узнайте у [@userinfobot](https://t.me/userinfobot))

### 5. Подготовка Telegram канала

Бот должен быть админом в Telegram канале с правами:
- ✅ Читать сообщения
- ✅ Редактировать сообщения
- ✅ Удалять сообщения
- ✅ Управлять пользователями
- ✅ Приглашать пользователей

### 6. Подготовка Max

Добавьте Max бота в нужный канал с правами на отправку сообщений.

## 🚀 Запуск

```bash
python bot/main.py
```

Бот будет запущен и начнёт слушать сообщения в добавленных Telegram каналах.

## 📡 Использование

### Админ-панель

Откройте админ-панель командой:
```
/admin
```

### Управление каналами

1. Выберите пункт **📡 Каналы**
2. Нажмите **➕ Добавить пару**
3. Введите username или ID Telegram канала
4. Введите ID канала в Max
5. Подтвердите добавление

Для каждого канала можно:
- ✅ Включить/отключить ретрансляцию
- 🗑 Удалить пару каналов

### Слова-исключения

1. Выберите **🚫 Слова-исключения**
2. Выберите тип:
   - **🌍 Глобальные** - применяются ко всем каналам
   - **📡 По каналу** - только для конкретного канала

3. Добавляйте слова через запятую

Если сообщение содержит слово-исключение (регистронезависимо), оно будет пропущено.

### Статистика

1. Выберите **📊 Статистика**
2. Просмотрите:
   - Общее количество пересланных и пропущенных постов
   - Статистику по каждому каналу
   - Дату последней пересылки

## 🗄️ Структура базы данных

### channel_pairs
Пары связанных каналов TG и Max:
- `id` - уникальный идентификатор
- `tg_channel_id` - ID Telegram канала
- `tg_channel_username` - username Telegram канала
- `max_channel_id` - ID канала в Max
- `is_active` - статус ретрансляции (1 - активна, 0 - отключена)

### excluded_words
Слова-исключения:
- `id` - уникальный идентификатор
- `word` - само слово
- `channel_pair_id` - ID пары каналов (NULL = глобальное слово)

### forwarded_posts
История переслано сообщений:
- `tg_message_id` - ID сообщения в TG
- `tg_channel_id` - ID канала в TG
- `forwarded_at` - дата и время пересылки

### skipped_posts
История пропущено сообщений:
- `tg_message_id` - ID сообщения в TG
- `tg_channel_id` - ID канала в TG
- `reason` - причина пропуска
- `skipped_at` - дата и время пропуска

## 📝 Логирование

Все события логируются в консоль с уровнями:
- **INFO** - стандартные операции (пересылка, добавление каналов)
- **ERROR** - ошибки при обработке сообщений или работе с API

Пример логов:
```
2024-06-29 10:15:23,456 - bot.tg_handler - INFO - Message 12345 from -100123456789 forwarded to 999
2024-06-29 10:16:45,789 - bot.db - INFO - Added channel pair: TG -100123456789 -> MAX 999
```

## ⚙️ Как работает ретрансляция

1. **Получение поста**: бот получает новое сообщение из Telegram канала
2. **Проверка дубля**: проверяется таблица `forwarded_posts` на дубликаты
3. **Фильтрация**: текст проверяется на слова-исключения
4. **Обработка содержимого**:
   - Текст и подпись (caption) сохраняются полностью
   - Custom emoji заменяются на unicode эмодзи
   - Медиа скачиваются из TG и загружаются в Max
   - Кнопки преобразуются в формат Max
   - Медиагруппы (альбомы) отправляются одним сообщением
5. **Отправка**: сформированное сообщение отправляется в Max
6. **Логирование**: информация о пересылке записывается в БД

## 🐛 Обработка edge cases

- ✅ Посты без текста (только медиа)
- ✅ Медиагруппы (альбомы из нескольких фото/видео)
- ✅ Форварды из других каналов
- ✅ Сообщения с кнопками и меню
- ✅ Все ошибки логируются и не приводят к падению бота

## 🔒 Безопасность
